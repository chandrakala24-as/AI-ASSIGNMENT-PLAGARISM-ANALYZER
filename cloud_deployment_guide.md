# PlagCheck AI Cloud Deployment Guide

This guide walks you through deploying the PlagCheck AI application across three distinct cloud services: **MongoDB Atlas** for the database, **Render** for the FastAPI backend, and **Vercel** for the frontend.

---

## Step 1: Deploy Database on MongoDB Atlas

1. **Create an Account:** Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas/register) and sign up for a free account.
2. **Create a Cluster:** 
   - Click **"Build a Database"**.
   - Select the **M0 Free** shared tier.
   - Choose a cloud provider (e.g., AWS) and region closest to you.
   - Click **"Create Cluster"**.
3. **Database Access (User):**
   - In the "Quickstart" or "Database Access" menu, create a database user (e.g., username `plagcheck_user` and an autogenerated password). **Save this password**.
4. **Network Access (IP Whitelist):**
   - Go to "Network Access" in the sidebar.
   - Click **"Add IP Address"** and select **"Allow Access from Anywhere"** (`0.0.0.0/0`) since Render's outgoing IPs can change.
5. **Get Connection String:**
   - Go to "Database" -> Click **"Connect"** on your cluster.
   - Select **"Drivers"**.
   - Copy the connection string. It will look like this: `mongodb+srv://plagcheck_user:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority`
   - Replace `<password>` with the password you saved earlier. This is your `MONGO_URI`.

---

## Step 2: Deploy Backend on Render

1. **Create an Account:** Go to [Render](https://render.com/) and sign up.
2. **Create a New Web Service:**
   - Click **"New +"** and select **"Web Service"**.
   - Connect your GitHub account and select your repository.
3. **Configure the Service:**
   - **Name:** `plagcheck-ai-backend` (or similar).
   - **Environment:** `Python 3`.
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`
4. **Set Environment Variables:**
   - Scroll down to "Environment Variables" and add:
     - **Key:** `MONGO_URI`
     - **Value:** *(Paste your MongoDB Atlas connection string here)*
5. **Deploy:** Click **"Create Web Service"**.
   - *Note:* It will take a few minutes to build and download the heavy BERT models. Keep an eye on the logs.
   - Once deployed, copy your Render URL (e.g., `https://plagcheck-ai-backend.onrender.com`).

---

## Step 3: Deploy Frontend on Vercel

*Before doing this, open the `vercel.json` file in your repository and replace `<YOUR_RENDER_APP_NAME>` with your actual Render backend URL (excluding the `https://` and `/api/...` parts).* 
*Example: If your Render URL is `https://plagcheck-ai.onrender.com`, the destination should be `https://plagcheck-ai.onrender.com/api/$1`.*

1. **Commit and Push:** Ensure your updated `vercel.json` is committed and pushed to your GitHub repo.
2. **Create an Account:** Go to [Vercel](https://vercel.com/) and sign in with GitHub.
3. **Import Project:**
   - Click **"Add New..."** -> **"Project"**.
   - Import your repository.
4. **Configure Project:**
   - **Framework Preset:** Leave as "Other".
   - **Root Directory:** Leave it as `./` (the root). Vercel will automatically serve your HTML files and apply the `vercel.json` rules.
5. **Deploy:** Click **"Deploy"**.
6. **Access App:** Once Vercel finishes, visit the provided URL. The `vercel.json` will seamlessly route all requests made to `/api/...` from your frontend directly to your Render backend.

> [!TIP]
> **Why `vercel.json`?** 
> By proxying the API requests through Vercel, you avoid Cross-Origin Resource Sharing (CORS) issues and do not need to hardcode absolute URLs into your JavaScript files.
